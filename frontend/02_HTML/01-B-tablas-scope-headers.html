scope vs headers

Ambos conectan celdas de datos con sus encabezados para que los lectores de pantalla anuncien el contexto. La diferencia es cuándo usar cada uno.

1. scope="col" — Encabezados de columna

El caso más simple. Los encabezados están arriba y mandan hacia abajo.
El lector de pantalla al llegar a "28" anuncia: "Edad: 28".

scope="col" — Encabezados de columna
Nombre Edad Ciudad
Ana 28 Córdoba
Luis 35 Rosario

2. scope="row" — Encabezados de fila

Los encabezados están a la izquierda y mandan hacia la derecha.
El lector al llegar a "22°" anuncia: "Lunes, Máxima: 22°".

scope="row" — Encabezados de fila
Día Mínima Máxima
Lunes 14° 22°
Martes 15° 25°
Miércoles 12° 19°

3. scope="col" + scope="row" — Combinados

Encabezados arriba Y a la izquierda. La celda recibe contexto de ambos.
El lector al llegar a "$150" anuncia: "Ventas, Febrero: $150".

scope combinado — Ventas por departamento
Enero Febrero Marzo
Ventas $100 $150 $120
Marketing $80 $90 $85
IT $200 $210 $220
Total $380 $450 $425

4. scope="colgroup" y scope="rowgroup"

Para encabezados que abarcan un GRUPO de columnas o filas (usan colspan o rowspan).
Hasta acá llega scope. Para tablas más complejas, se necesita headers.

scope colgroup/rowgroup — Horario de clases
Mañana Tarde
8:00 10:00 14:00 16:00
Lunes HTML CSS JavaScript React
Martes Node.js Express MongoDB Proyecto

⚠ El límite de scope

scope funciona cuando la tabla es regular: una fila de encabezados arriba, una columna a la izquierda, sin celdas fusionadas que rompan la estructura.

Cuando la tabla tiene colspan / rowspan complejos, múltiples niveles de encabezados, o celdas que dependen de encabezados que no están en su misma fila ni columna, scope se vuelve ambiguo y el lector de pantalla no puede inferir las relaciones.

Ahí es donde entra headers.

5. headers — La misma tabla pero explícita

Cada celda dice exactamente cuáles son sus encabezados usando headers="id1 id2".
Más trabajo, pero cero ambigüedad.

headers — Ventas por departamento (explícito)
Enero Febrero Marzo
Ventas $100 $150 $120
Marketing $80 $90 $85
IT $200 $210 $220

Comparación: Esta tabla es idéntica a la del ejemplo 3. Con scope alcanzaba. Pero headers también funciona. La diferencia se nota en las tablas complejas de abajo.

6. headers — Tabla con colspan (scope NO alcanza)

"Primer Semestre" abarca Q1 y Q2. ¿A cuál aplica scope="col"? A las dos, pero scope no puede decir "aplico a las próximas 2 columnas" de forma confiable. headers sí puede.

headers con colspan — Presupuesto semestral
Primer Semestre Segundo Semestre
Q1 Q2 Q3 Q4
Ventas $100 $150 $120 $180
Gastos $80 $90 $85 $95
Ganancia $20 $60 $35 $85

7. headers — Tabla con rowspan (scope NO alcanza)

"Electrónica" abarca 2 filas con rowspan. scope="row" solo aplica a una fila. headers permite asociar "Tablet" con "Electrónica" explícitamente.

headers con rowspan — Productos por categoría
Categoría Producto Precio Stock
Electrónica Smartphone $800 150
Tablet $500 80
Ropa Remera $25 500
Campera $120 60

8. headers — Tabla compleja (colspan + rowspan + 3 niveles)

Esta tabla tiene todo: colspan en los encabezados, rowspan en las filas, y tres niveles de encabezados. Sin headers, un lector de pantalla no podría darle contexto a cada celda.

Informe de ventas — Regional por canal y trimestre
Online Local
Q1 Q2 Q1 Q2
Norte Ventas $50 $60 $30 $35
Devoluciones $5 $3 $2 $4
Sur Ventas $40 $55 $25 $30
Devoluciones $3 $2 $1 $2

El lector de pantalla al llegar a "$50" anuncia:
"Norte, Ventas, Online, Q1: $50"
porque headers="r-norte r-norte-vta c-online c-on-q1" le da los 4 niveles de contexto: región, tipo, canal, trimestre.

Resumen — ¿Cuándo usar cuál?

scope vs headers — Comparación
Característica scope headers
Se pone en <th> <td> o <th>
Necesita IDs No Sí (en cada <th>)
Relación Implícita (por posición) Explícita (por ID)
Tablas simples Perfecto Funciona pero es innecesario
Tablas con colspan Limitado / ambiguo Perfecto
Tablas con rowspan Limitado / ambiguo Perfecto
Múltiples niveles No soporta bien Perfecto (sin límite)
Esfuerzo de código Poco (solo agregar scope) Mucho (IDs + headers en cada celda)
Recomendación Usar siempre en tablas simples Usar cuando scope no alcanza

Regla práctica

  1. ¿La tabla tiene una sola fila de encabezados arriba y/o una columna a la izquierda, sin colspan ni rowspan?
    → Usá scope. Es suficiente.
  2. ¿La tabla tiene colspan, rowspan, o múltiples niveles de encabezados?
    → Usá headers con id. Es la única forma confiable.
  3. ¿No estás seguro?
    → Usá headers. Nunca falla. Es más trabajo pero cubre todo.
