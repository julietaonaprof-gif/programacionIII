Guía Completa de <audio> en HTML

La etiqueta <audio> permite reproducir sonido directamente en la página web sin plugins externos.
📖 MDN — <audio>

1. Audio básico con controles

El atributo controls muestra el reproductor nativo del navegador (play/pausa, barra de progreso, volumen).
Sin controls, el audio existe pero el usuario no puede interactuar. 

<audio controls>
  <source src="audio/cancion.mp3" type="audio/mpeg">
  Tu navegador no soporta la etiqueta audio.
</audio>

2. Múltiples formatos con <source>

Se ponen varios <source> con diferentes formatos. El navegador usa el primero que soporte e ignora el resto.
📖 MDN — <source> 

<audio controls>
  <source src="audio/cancion.ogg" type="audio/ogg">
  <source src="audio/cancion.mp3" type="audio/mpeg">
  <source src="audio/cancion.wav" type="audio/wav">
  Tu navegador no soporta audio.
</audio>

3. Usando src directo (sin source)

Si solo tenés un formato, podés poner el src directamente en la etiqueta <audio> sin usar <source>. 

<audio controls src="audio/cancion.mp3">
  Tu navegador no soporta audio.
</audio>

4. Reproducción automática — autoplay

autoplay intenta reproducir el audio automáticamente al cargar la página.
IMPORTANTE: la mayoría de navegadores bloquean el autoplay con sonido. Solo funciona si el audio tiene también muted, o si el usuario ya interactuó con la página antes. 

<!-- Funciona: autoplay + muted -->
<audio controls autoplay muted>
  <source src="audio/notificacion.mp3" type="audio/mpeg">
</audio>

<!-- BLOQUEADO por el navegador: autoplay sin muted -->
<audio controls autoplay>
  <source src="audio/notificacion.mp3" type="audio/mpeg">
</audio>

5. Repetir en bucle — loop

loop hace que el audio se repita automáticamente cuando llega al final. Útil para música de fondo o ambientación. 

<audio controls loop>
  <source src="audio/ambiente.mp3" type="audio/mpeg">
</audio>

<!-- Música de fondo automática silenciada en loop -->
<audio autoplay muted loop>
  <source src="audio/fondo.mp3" type="audio/mpeg">
</audio>

6. Precarga — preload

preload le indica al navegador qué tanto del audio debería descargar anticipadamente al cargar la página.

preload="auto" — Descarga todo:



preload="metadata" — Solo metadatos (duración, etc.):



preload="none" — No descarga nada hasta que apretés play:

<audio controls preload="auto">...</audio>     <!-- Descarga todo -->
<audio controls preload="metadata">...</audio> <!-- Solo metadatos -->
<audio controls preload="none">...</audio>     <!-- Nada hasta play -->

7. Audio sin controles — Controlado por JavaScript

Sin controls, el audio es invisible. Se controla 100% desde JavaScript. Útil para efectos de sonido, notificaciones, o reproductores personalizados. 

<!-- Audio invisible (sin controls) -->
<audio id="efecto-click">
  <source src="audio/click.mp3" type="audio/mpeg">
</audio>

<!-- Botón que lo reproduce -->
<button onclick="document.getElementById('efecto-click').play()">
  Reproducir
</button>

8. Reproductor personalizado con JavaScript

Podés ocultar los controles nativos y crear los tuyos propios usando botones HTML y JavaScript. 

<audio id="mi-audio">
  <source src="audio/cancion.mp3" type="audio/mpeg">
</audio>

<button onclick="document.getElementById('mi-audio').play()">Play</button>
<button onclick="document.getElementById('mi-audio').pause()">Pausa</button>
<button onclick="document.getElementById('mi-audio').currentTime = 0">Reiniciar</button>
<button onclick="document.getElementById('mi-audio').volume += 0.1">+Vol</button>
<button onclick="document.getElementById('mi-audio').playbackRate = 2">x2</button>

9. Audio desde una URL externa

El src puede ser un archivo local o una URL externa. Funciona con cualquier URL pública que apunte a un archivo de audio. 

<audio controls preload="none" crossorigin="anonymous">
  <source src="https://www.soundhelix.com/examples/mp3/SoundHelix-Song-1.mp3"
          type="audio/mpeg">
</audio>

crossorigin="anonymous": pide el audio sin enviar cookies. Necesario para manipular audio externo con JavaScript (Web Audio API).
📖 MDN — crossorigin

Cuando tenés varios audios en la misma página, es importante usar preload="none" para no descargar todos a la vez.

11. Accesibilidad en audio

El audio no es accesible por sí solo para personas sordas o con dificultades auditivas. Hay que proveer alternativas.

Ver transcripción del Episodio 1

Bienvenidos al primer episodio de nuestro podcast sobre desarrollo web. Hoy vamos a hablar sobre HTML, el lenguaje de marcado que forma la base de toda página web. HTML significa HyperText Markup Language... [transcripción completa del audio] 

<audio controls
       aria-label="Episodio 1: Intro a HTML, 15 min"
       aria-describedby="transcripcion">
  <source src="audio/ep01.mp3" type="audio/mpeg">
</audio>

<!-- Transcripción colapsable -->
<details id="transcripcion">
  <summary>Ver transcripción</summary>
  <p>Texto completo del audio...</p>
</details>

Resumen de atributos de <audio>

Todos los atributos de la etiqueta audio
Atributo Valores Descripción
src URL Ruta al archivo de audio (alternativa a usar <source>)
controls (booleano) Muestra los controles nativos (play, pausa, volumen)
autoplay (booleano) Reproduce automáticamente (bloqueado sin muted)
muted (booleano) Arranca silenciado
loop (booleano) Se repite al terminar
preload auto | metadata | none Cuánto descargar anticipadamente
crossorigin anonymous | use-credentials Configuración CORS para audio externo

Formatos de audio soportados
Formato MIME Type Chrome Firefox Safari Notas
MP3 audio/mpeg Sí Sí Sí El más compatible. Usar como fallback siempre.
OGG Vorbis audio/ogg Sí Sí No Libre de patentes. No funciona en Safari.
WAV audio/wav Sí Sí Sí Sin compresión. Archivos muy pesados.
WebM audio/webm Sí Sí No Buena compresión. No funciona en Safari.
AAC audio/aac Sí Sí Sí Usado por Apple (iTunes, iPhone).
FLAC audio/flac Sí Sí Sí Sin pérdida. Archivos pesados.

Buenas prácticas

  1. Siempre poner controls a menos que controles el audio con JavaScript. Un audio sin controles y sin JS es inaccesible.
  2. Proveer al menos MP3 como formato. Es el que funciona en todos los navegadores.
  3. Usar preload="none" si tenés varios audios en la página para ahorrar ancho de banda.
  4. No usar autoplay con sonido. Los navegadores lo bloquean y es molesto para el usuario.
  5. Siempre ofrecer una transcripción para accesibilidad (personas sordas o con dificultades auditivas).
  6. Usar aria-label para describir el contenido del audio a lectores de pantalla.
  7. Incluir texto de fallback entre las etiquetas por si el navegador no soporta audio (raro hoy).
  8. Especificar type en cada <source> para que el navegador sepa qué formato es sin descargarlo.
